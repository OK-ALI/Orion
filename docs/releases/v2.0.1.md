# Orion X Music Planet v2.0.1

v2.0.1 is a maintenance and reliability update focused on repairing Cinema streaming sources, updating provider embed domains, and hardening ad-blocking across embedded webview sessions.

## Highlights

### Cinema Streaming Sources & Domain Migration
- **VidSrc domain migration:** Updated primary VidSrc embed endpoints from the deprecated `vidsrc.to` domain to VidSrc's official updated mirror `vsembed.su`.
- **Videasy domain update:** Updated Videasy primary endpoints from `player.videasy.net` to `player.videasy.to` in alignment with active provider APIs.
- **VsEmbed experimental update:** Updated VsEmbed experimental adapter to `vsembed.su` with correct origin contracts.
- **111Movies embed endpoint fix:** Corrected 111Movies experimental adapter from the `111movies.com` landing page to the dedicated `111movies.net` player endpoint.
- **Unreachable source quarantine:** Moved VidSrc CC (`vidsrc.cc`) to disabled sources due to persistent HTTP 522 upstream host failures.

### Ad-Blocking & Session Security
- **Electron session Manager (`sessionManager.js`) additions:** Expanded `BLOCKED_HOSTS` with over 15 tracking vectors, including `users.videasy.net` and `users.videasy.to` tracking pixels, `analytics.vixcloud.co`, Google font/analytics trackers embedded within third-party frames, and VidSrc-specific ad redirect hosts (`qj.asokapygmoid.com`, `hfriendsof.net`, `howletoperation.com`).
- **DOM-level overlay removal (`playerWebviewPreload.js`):** Enhanced webview DOM cleaner to neutralize `div`-based click-hijack overlays (`z-index: 2147483647`), transparent full-screen anchor popunders (`a[target='_blank']`), and common ad-container class patterns before user interaction.
- **Clean origin whitelisting:** Removed `users.videasy.net` tracking host from Videasy's `requiredRequestOrigins` manifest so tracking pixels are blocked rather than passed through.
- **Webview postMessage bridge:** Whitelisted `https://vsembed.su` in `PLAYER_EVENT_ORIGINS` to allow sanitized progress tracking and playback telemetry.

## Validation

The release has been validated through:
- Automated source verification scanning all active provider endpoints for HTTP status, content signatures, and CORS headers.
- Electron webview preload and postMessage event contract tests.
- `npm.cmd run check` suite passing clean builds, static checks, binding validation, and test suites.

## Compatibility

Existing user data, Google account backup states, local library indexes, custom provider keys, and saved source preferences remain 100% compatible.
